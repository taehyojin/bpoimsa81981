# KAGACHI-SAMA CANON AUDIT RULES

Version 1.0

Purpose:

The purpose of this audit is to create a canon-accurate reference for Surviving Kagachi-sama while minimizing contradictions with the original visual novel.

The goal is not speed.

The goal is accuracy.

All findings should be traceable to the source material.

---

# SOURCE HIERARCHY

Primary Source:

Kagachi_Master_Clean_v2.md

Supplementary Source:

Original .ks files

The master archive is the primary text used during auditing.

The .ks files exist primarily to verify:

* filename
* file boundaries
* file ordering
* formatting anomalies

Audits should not be based on memory.

Audits should not be based on previous summaries.

Audits should be based on the source text currently under review.

---

# ENTITY DISCOVERY RULE

At the end of every file audit, perform an explicit Entity Discovery pass.

The auditor must review the file a second time and identify any newly introduced:

* People
* Locations
* Organizations
* Family Relationships
* Village Roles / Titles
* Named Objects
* Named Rituals

regardless of how briefly they appear.

Every newly introduced entity must be recorded.

Even a single line of dialogue can introduce a sequel-relevant entity.

Examples:

People:

* Chikusa
* Gonzo
* Reito Sudo

Locations:

* Sudo Family Barn
* Nagataro House

Relationships:

* Chikusa = Takahiko's mother
* Ayaka = Takahiko's first love
* Chotaro = Takahiko's father

Village Roles:

* Kagachi-sama
* Shrine caretaker
* Village elder

Named Rituals:

* Itch treatment ritual
* Harvest ceremony

If an entity is mentioned but its role is uncertain, record it anyway and mark the certainty level.

Example:

Chikusa
Role:
Likely Takahiko's mother.

Evidence:
Takahiko identifies Chikusa as "my mom."

Certainty:
High.

The purpose of this rule is to prevent important canon entities from being omitted simply because they appear briefly or outside the main focus of a scene.

---

# FACTS VS INTERPRETATION

Every audit should distinguish between:

Verified Facts

and

Interpretation

Verified Facts:

Must be directly supported by the source.

Examples:

* locations
* participants
* dialogue
* events
* timeline information
* revelations
* family relationships
* village customs

Interpretation:

May include observations such as:

* foreshadowing
* thematic relevance
* character implications
* sequel relevance

Interpretation must never be presented as fact.

# FILE NUMBERING RULE

Treat:

kagXXX_YYh

as:

Scene YY

not:

A separate branch, alternate scene, or missing numbered file.

The trailing "h" exists only to indicate that the file contains H-content.

It does not create a new scene number.

Examples:

kag000_01
kag000_02h
kag000_03

should be interpreted as:

Scene 1
↓
Scene 2 (H)
↓
Scene 3

not:

Scene 1
↓
Scene 2
↓
Scene 2H
↓
Scene 3

During audits:

- Do not assume a missing non-H counterpart exists.
- Do not report missing numbered files solely because an H-marked file occupies that number.
- Treat H-files as part of the main narrative sequence.
- Evaluate their story-state changes with the same importance as non-H scenes.

The "h" suffix indicates content type only.

It does not indicate a separate timeline, route, branch, or numbering sequence.

# SCENE SUMMARY RULES

Every significant scene should include:

Scene Name

Location

What Happens

Result / End State

Location is mandatory.

Scene names should be recognizable.

Examples:

* Barn Cheating Scene
* Rope-and-Yam Ordeal
* River Outing
* Festival Normalization Scene
* Breakfast at Tomikura House

Avoid generic labels.

Bad:

"Important scene"

Good:

"Nagataro House Rope-and-Yam Ordeal"

---

# H-SCENE AUDIT RULES

H-scenes are not to be summarized as erotic content.

They are to be summarized as story-state changes.

For every H-scene include:

Scene Name

Location

Framing

Who Is Involved

What Happens

Important Objects / Ordeals

Result / End State

Interpretation

The objective is to record:

* what happened
* why it matters
* where the characters end up afterward

---

# FRAMING RULE

Every H-scene must explicitly identify its framing.

Possible framings:

* Present-day event
* Memory
* Recurring dream based on memory
* Dream
* Fantasy
* Hallucination
* Unclear

Never guess.

If the source does not establish framing clearly:

State:

"Framing unclear."

---

# OBJECT RULE

If a non-sexual object is central to the scene, it must be named.

Examples:

* Yam
* Rope
* Eels
* Farm equipment
* Ritual objects
* Festival implements

These objects are part of scene identity and are often important to later interpretation.

If fans would identify a scene by the object, the object belongs in the summary.

---

# LOCATION RULE

Locations must be recorded whenever possible.

Examples:

* Tomikura House
* Nagataro House
* Sudo Family Barn
* Shiranawa Village
* Riverbank
* Festival Grounds

Location is frequently important to the progression of corruption, normalization, and character development.

---

# MTL INTERPRETATION RULE

The translation contains known MTL artifacts.

During auditing:

Use common sense and surrounding context when interpreting ambiguous wording.

Do not create dedicated translation-correction sections.

The goal is to accurately capture the intended scene content rather than document translator mistakes.

---

# NO MEMORY RULE

If a finding cannot be verified from the source currently under review:

Do not report it as fact.

Do not reconstruct scenes from memory.

Do not fill gaps with assumptions.

When uncertain:

Return to the source.

---

# ULTIMATE AUDIT QUESTION

For every scene ask:

What literally happened?

Why does it matter?

Where do the characters end up afterward?

If a future Surviving Kagachi-sama scene could contradict the answer to one of those questions, the information belongs in the audit.

Accuracy is more important than speed.
